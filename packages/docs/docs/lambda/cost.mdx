在本页面中，介绍了一些优化使用 Remotion Lambda 成本的策略。

## 降低并发

Lambda 函数的大量计算时间用于启动 Lambda 函数、打开浏览器、上传和下载资产。通过使用更少的 Lambda 函数，可以减少产生的开销，最终会导致更低的成本，但也会导致渲染速度变慢。请查看 [Lambda 并发](/docs/lambda/concurrency) 页面获取更多信息。

## 使用更便宜的区域

并非所有的 AWS 区域成本相同。查看 [AWS Lambda](https://aws.amazon.com/lambda/pricing/) 定价表，以查看您是否使用了更昂贵的区域。

## 预先计算数据

考虑您的计算是否会在每次调用 Lambda 函数时运行。如果可能的话，预先计算数据一次，并将其作为 [输入属性](/docs/passing-props#passing-input-props-in-the-cli) 传递给渲染。

## 减少内存

减少内存将线性降低成本。在部署 Lambda 函数时尝试不同的内存值，并查看您可以将其设置多低而不会导致崩溃。

## 使渲染更高效

使您的渲染更高效也会降低使用 Lambda 的成本。请参阅 [一般性能提示](/docs/performance)。
