如果您收到错误消息：

```
TooManyRequestsException: Rate Exceeded.
```

在调用 Lambda 函数时，这意味着您的并发限制已达到。

- **并发限制**：每个区域每个帐户可以同时运行的 Lambda 函数的最大数量。
- **突发限制**：在 10 秒内并发增加的最大数量（突发限制为 1000）

## 默认并发限制

默认情况下，并发限制为每个区域 1000 个函数，但在某些区域，突发限制仅为 500，从某种程度上限制了您可以使用的规模。

## 异常情况：使用 AWS Lambda 的新帐户

根据 AWS 的说法，“一些帐户”在首次使用 AWS Lambda 时可能会获得非常低的并发限制，例如 10。在这种情况下，请通过 AWS 控制台或 Remotion CLI（见下文）增加限制。

## 查看您的限制

要查看您的限制，请运行

```
npx remotion lambda quotas
```

:::note
如果出现权限错误，请在 [设置指南](/docs/lambda/setup) 中重复用户策略步骤，并在 AWS 控制台中更新您的用户策略文件。
:::

## 请求增加

您可以在 [`https://console.aws.amazon.com/servicequotas/home`](https://console.aws.amazon.com/servicequotas/home) 或使用 [Remotion CLI](/docs/lambda/cli/quotas) 请求配额增加：

```
npx remotion lambda quotas increase
```

:::note
这仅适用于 AWS 根帐户，而不适用于组织的子帐户。您仍然可以通过控制台请求增加。
:::

[在此处查看](/docs/lambda/limits#if-aws-asks-you-for-the-reason) 如果 AWS 询问您请求增加的原因的默认答案。
