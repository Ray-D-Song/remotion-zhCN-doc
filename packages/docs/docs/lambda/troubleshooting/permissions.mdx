如果您收到错误消息，指出您的 AWS 权限存在问题，请阅读本页面以获取故障排除的想法。

## Node.JS APIs 不读取 `.env` 文件

如果您使用 Node.JS APIs，请注意它们不会自动读取 .env 文件。这意味着您需要加载 .env 文件或手动设置环境变量。

## 用户和角色权限混淆

有两个策略文件，一个用于 AWS 用户，一个用于 AWS 角色。这些策略不同，确保您已正确分配了两者。

## 设置应用的时间不够

可能需要几分钟才能使您输入到 AWS 的策略生效。有时等待 2-3 分钟可以解决问题。

## 所需权限已更改

较新的 Remotion Lambda 版本可能需要额外的权限。我们将在 [更新日志](https://github.com/remotion-dev/remotion/releases) 中注明此事。确保您在 AWS 中更新策略。

## 可能已应用其他 AWS 凭据

如果 AWS 环境变量加载失败，则可能已从其他位置（如 AWS CLI）加载了其他凭据。记录环境变量以确保您加载了正确的凭据。

## 使用验证命令

使用 [`npx remotion lambda policies validate`](/docs/lambda/cli/policies) 命令验证用户策略。请注意，这仍可能意味着角色策略设置错误或在使用 Node.JS APIs 时环境变量未加载。
