显示当前应用于您的帐户和区域的 AWS Lambda 并发和突发限制。您可以使用 `increase` 命令向 AWS 发送请求，要求它们增加配额。

- [`quotas`](#quotas)
- [`quotas increase`](#quotas-increase)

## `quotas`

打印当前应用于您的 AWS 帐户和区域的限制。

```
npx remotion lambda quotas
```

<details>
<summary>显示示例输出</summary>
<pre>
Region = us-east-1

并发限制: 1000 - 建议增加！
一个将其增加到 5000 的请求正在等待：
https://us-east-1.console.aws.amazon.com/support/home#/case/?displayId=9742781451
可以同时执行的 Lambda 函数的最大数量。
运行 `npx remotion lambda quotas increase` 以请求 AWS 增加您的限制。

</pre>
</details>

### `--region`

应打印配额的区域。

## `quotas increase`

创建一个 AWS 支持请求，以增加您的帐户上的并发限制，以及可能存在于您的帐户上的配额增加请求。

```
npx remotion lambda quotas increase
```

:::note
这仅适用于 AWS 根帐户，而不适用于组织的子帐户。您仍然可以通过控制台请求增加。
:::

### `--region`

应增加配额的区域。

### `--yes`

跳过请求确认。

### `--force`

即使是第二次请求，也会请求增加。如果您没有指定消息，那么您获得批准的可能性更小，建议使用控制台进行进一步增加。
