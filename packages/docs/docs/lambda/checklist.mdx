您已经使用 Remotion Lambda 实现了您的解决方案，并准备将项目推向世界。恭喜！
在您上线之前，请通过此清单确保 Lambda 运行稳定。

### 优化内存

给 Lambda 函数添加过多内存可能会使渲染成本更高。将函数的内存减少 25% 也会使成本降低 25%！多次重新部署函数，并尽可能降低内存大小，直到您觉得在低成本和视频可靠渲染之间找到了最佳平衡点。

### 最大文件大小

Lambda 受限于最大输出文件大小约为[磁盘空间的一半](/docs/lambda/disk-size)。调整 Lambda 的磁盘空间参数以适应您希望支持的最大视频长度。测试输出文件的文件大小，并确保它们不会超过限制。
如果您的视频基于用户输入，请防止用户渲染非常长的视频，这将超出 Remotion Lambda 中可用的空间。

### 权限

确保您的 AWS 用户只具有所需的权限，并将您的凭据存储为环境变量。查看 [权限](/docs/lambda/permissions) 页面，了解最低权限要求是什么。

### 选择正确的并发数

如果您使用 [`framesPerLambda`](/docs/lambda/rendermediaonlambda#framesperlambda) 选项，请确保对于每个要渲染的视频，参数设置在允许的范围内（每次渲染不超过 200 个 Lambda 函数）。参见：[并发性](/docs/lambda/concurrency)

### 存储桶隐私

默认情况下，渲染的视频在您的存储桶中是公开可访问的。如果需要，可以在 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda) 和 [`renderStillOnLambda()`](/docs/lambda/renderstillonlambda) 中使用 `privacy` 设置使渲染私有。

### 速率限制

考虑到用户可能会调用许多视频渲染，这将反映在您的 AWS 账单上，请实现速率限制器，以防止恶意行为者渲染许多视频。



### 超时

您的 Lambda 函数的默认超时时间为 <DefaultTimeout /> 秒，这应该足够了，因为视频渲染是大规模并行的。但在这里，也要测量和调整超时时间以满足您的需求，并确保它足够长，以确保您的视频渲染不会失败。



### 有效的公司许可证

拥有超过 3 人的公司需要购买云渲染席位才能使用 Remotion Lambda。在启动之前，请熟悉 [许可证](https://github.com/remotion-dev/remotion/blob/main/LICENSE.md) 并 [购买必要的云席位](https://www.remotion.pro/)。
