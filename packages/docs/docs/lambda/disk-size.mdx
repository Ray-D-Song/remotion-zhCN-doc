默认情况下，每个 Lambda 函数都带有一个临时磁盘大小：

| Remotion 版本 | 默认值       |
|------------------|---------------|
| &lt;5.0.0        | 2048MB        |
| &gt;=5.0.0       | 10240MB       |

增加磁盘空间将允许更长的视频，并加快渲染速度，因为 Chrome 可以访问更多的磁盘缓存。  
磁盘。  

增加更多的磁盘空间几乎不会对渲染成本产生影响（成本增加不到1%）。  
因此，我们建议将磁盘大小设置为最大可能值：10240MB。

## 大致最大视频长度

请注意，我们建议将磁盘大小设置为最大可能值：10240MB。

| 磁盘大小         | 大致最大视频长度 |
| ----------------- | -------------------------------- |
| 512 MB            | 8 分钟 - 1080p                    |
| 1024 MB           | 16 分钟 - 1080p                   |
| 2048 MB           | 32 分钟 - 1080p                   |
| 4096 MB           | 1 小时 4 分钟 - 1080p                  |
| 8192 MB           | 2 小时 8 分钟 - 1080p                  |
| 10240 MB          | 2 小时 40 分钟 - 1080p                 |

> 这些是大致值，可能与您的情况不完全匹配。视频输出大小取决于视频内容和音频。测量并找到最适合您的值。

## 设置磁盘大小

- 使用 [`deployFunction()` 的 `diskSizeInMb` 选项](/docs/lambda/deployfunction#disksizeinmb) 在部署时设置磁盘大小。
- 如果使用 `remotion lambda functions deploy` 命令，可以使用 [`--disk`](/docs/lambda/cli/functions) 标志。

## 定价

使用更多的磁盘空间会略微增加成本。  
将磁盘大小设置为最大可能值：10240MB 的成本增加不到1%。  
请查看 [Lambda 定价页面](https://aws.amazon.com/lambda/pricing/) 的 "Lambda 临时存储定价" 部分了解定价。  
[`estimatePrice()`](/docs/lambda/estimateprice) API 也会考虑磁盘大小。
