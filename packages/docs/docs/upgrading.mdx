## 手动升级 Remotion

要将 Remotion 升级到最新版本，所有 `@remotion/*` 包和 `remotion` 必须更新到相同的版本。

编辑 `package.json` 中所有包的版本号。
删除 `package.json` 中版本号前面的 `^`，以强制使用您指定的确切版本。

## 破坏性更改

Remotion 遵循语义化版本控制。
这意味着如果版本号的第一个数字相同，您可以升级而且您的代码是向后兼容的。

示例：如果您使用的是 4.0.0 版本，您可以升级到 4.1.100 而无需更改您的代码。
但是，要升级到 5.0 版本，您需要遵循[迁移指南](/docs/5-0-migration)。

破坏性更改规则的例外是标记为实验性的 API。

## 稳定版本

我们维护一个包含 Remotion 最新稳定版本的存储库，供需要更高稳定性水平的客户使用。
客户可以在 [remotion.pro](https://remotion.pro) 门户获取对该存储库的访问权限。
