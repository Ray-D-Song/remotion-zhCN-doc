由于 JavaScript 代码正在执行，可能会发生由于异常而导致渲染失败的情况。以下是用于排除问题的一般提示。

<Step>1</Step> <strong>启用详细日志记录</strong>

通过启用更详细的日志记录，您的代码中的所有 `console.log` 语句将与其他调试信息一起显示。

从 CLI：在渲染命令中添加 [`--log=verbose`](/docs/cli/render#--log) 标志。  
从 Node.JS：在 `renderMedia()` 中添加 [`verbose: true`](/docs/renderer/render-media#verbose) 选项。

:::note
如果您多次看到一个日志，那是因为渲染被拆分为多个线程。暂时设置 `--concurrency=1` 以仅查看每个日志一次。
:::

<Step>2</Step> <strong>向项目添加日志</strong>

在您的代码中使用 `console.log` 来了解事物执行的顺序，并验证您对代码应该如何运行的假设。

<Step>3</Step> <strong>逐个删除组件</strong>

逐个删除组件直到视频为空。在哪个时刻错误消失？这可以帮助您确定导致渲染失败的组件。

<Step>4</Step> <strong>搜索问题和文档</strong>

检查 [GitHub 上的问题](https://github.com/remotion-dev/remotion/issues) 是否有其他人遇到类似的问题也是有帮助的。如果找到与您的问题匹配的问题，您可以在问题下添加评论以帮助社区排除问题。

还可以 [搜索文档](/search)，其中包含超过 300 页，包含许多常见问题的故障排除说明。

<Step>5</Step> <strong>寻求帮助</strong>

您可以在 GitHub 和 Discord 上寻求帮助。[继续阅读以了解如何获取帮助](/docs/get-help)！
