如果您收到以下错误消息：

```
无法启动浏览器进程！

故障排除: https://github.com/puppeteer/puppeteer/blob/main/docs/troubleshooting.md // 或指向此页面的URL
```

这意味着浏览器无法启动。最常见的原因是

## 缺少共享库

安装[适用于您的操作系统的缺失共享库](/docs/miscellaneous/linux-dependencies)。

## 错误的操作系统或架构

如果在 arm64 上运行 x64 浏览器二进制文件，或者反之亦然，您将收到此错误。  
如果浏览器是为不同的操作系统或 Linux 发行版编译的，它也可能不兼容。
确保您正在使用适用于您的操作系统和架构的正确二进制文件。

## 使用 `--log=verbose` 进行调试

在 `npx remotion render` 命令中添加 `--log=verbose` 标志。  
这将记录所有浏览器进程输出，帮助您调试问题。
