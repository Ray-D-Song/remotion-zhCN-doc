_自 v3.1 版本起可用_

您可以通过以下方式将视频渲染为 GIF：

- 在命令行中传递 `--codec=gif`
- 在 [`renderMedia()`](/docs/renderer/render-media)、[`stitchFramesToVideo()`](/docs/renderer/stitch-frames-to-video) 或 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda) 中设置 `codec: "gif"`。

## 降低帧率

通常 GIF 的帧率低于视频。为此，我们支持一个名为 `everyNthFrame` 的参数，其工作方式如下：

- 默认情况下，`everyNthFrame` 设置为 `1`：渲染帧 `0`、`1`、`2`、`3`、`4` 等。
- 假设 `everyNthFrame` 为 `2`，则仅渲染每第二帧：`1`、`3`、`5`、`7` 等。 一个 30FPS 的视频现在将成为一个 15FPS 的 GIF。
- 如果 `everyNthFrame` 为 `3`，则仅渲染每第三帧：`2`、`5`、`8`、`11` 等，模式继续。

`everyNthFrame` 支持：

- 在 [`renderFrames()`](/docs/renderer/render-frames#everynthframe)、[`renderMedia()`](/docs/renderer/render-media#everynthframe) 和 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda#everynthframe) 中
- 在本地使用 `--every-nth-frame=2` 的 CLI [本地](/docs/cli/render#--every-nth-frame) 或 [Lambda](/docs/lambda/cli/render#--every-nth-frame) 中
- 在配置文件中使用 [`setEveryNthFrame()`](/docs/config#seteverynthframe)。

## 更改循环次数

<!-- 更改标题将更改其他链接 -->

`numberOfGifLoops` 选项允许您设置循环次数如下：

- `null`（或在 CLI 中省略）表示无限循环 GIF。
- `0` 禁用循环。
- `1` 循环播放 GIF 一次（总共播放两次）
- `2` 循环播放 GIF 两次（总共播放三次）
- 依此类推。

`numberOfGifLoops` 选项可设置为：

- 在 CLI 中使用 `--number-of-gif-loops=0` 标志进行设置，[本地](/docs/cli/render#--number-of-gif-loops) 或 [Lambda](/docs/lambda/cli/render#--number-of-gif-loops) 中
- 在 [`stitchFramesToVideo()`](/docs/renderer/stitch-frames-to-video#numberofgifloops)、[`renderMedia()`](/docs/renderer/render-media#numberofgifloops) 和 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda#numberofgifloops) 中
- 在配置文件中使用 [`setNumberOfGifLoops()`](/docs/config#setnumberofgifloops)。

## 透明 GIF

要渲染透明 GIF，必须将 [`imageFormat`](/docs/renderer/render-media) 选项设置为 `"png"`。在 Remotion Studio 中，可以在 "Picture" 选项卡中设置。
