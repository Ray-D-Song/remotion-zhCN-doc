Remotion 允许[摄入](/docs/passing-props)、[验证](/docs/schemas)、[可视化编辑](/docs/visual-editing)和转换可能用于参数化视频的数据。

数据可能会[影响视频的内容](/docs/data-fetching)，或者[元数据](/docs/dynamic-metadata)如宽度、高度、持续时间或帧速率。

## 高级概述

Remotion 允许将[props](https://react.dev/learn/passing-props-to-a-component)传递给 React 组件。  
Props 是 React 的概念，采用 JavaScript 对象的形式。

为确定传递给视频的数据，执行以下步骤：

<Step>1</Step> <strong>默认 props</strong> 静态定义，以便在 Studio 中设计视频时不需要任何数据。<br/>

<ul>
<li>
默认 props 定义数据的形状。
</li>
<li>
可以定义并验证模式。
</li>
<li>
在没有数据的情况下，可以在 Remotion Studio 中编辑默认 props。
</li>
</ul>
<Step>2</Step> <strong>输入 props</strong> 可在渲染视频时指定，以覆盖默认 props。<br/>
<ul>
<li>
输入 props 与默认 props 合并，其中输入 props 优先级更高。
</li>
</ul>

<Step>3</Step> 使用 <strong><a href="/docs/data-fetching"><code>calculateMetadata()</code></a></strong>，可以对 props 进行后处理，并动态计算元数据。<br/>

<ul>
<li>
例如，如果将 URL 作为 prop 传递，可以获取该 URL 并将内容添加到 props 中。
</li>
<li>
此处还可以进行视频持续时间和其他元数据的异步计算。
</li>
</ul>
<Step>4</Step> <strong>最终 props</strong> 传递给 React 组件。
<ul>
<li>
组件可以根据 props 动态呈现内容。
</li>
</ul>

查看[这里](/docs/props-resolution)以获取解析过程的可视化解释和更多详细信息。
