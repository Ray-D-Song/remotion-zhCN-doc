如果您从任何 [Remotion 模板](/templates) 开始，除了 Next.js 和 Remix 模板之外，您目前拥有一个应用程序，您可以在其中启动 [Remotion Studio](/docs/terminology/studio)。

如果您现在想将您的项目转换为应用程序，请按照以下步骤操作。

<Step>1</Step> 使用以下模板之一搭建一个新项目：

- [Next.js（应用程序目录）](/templates/next)
- [Next.js（应用程序目录 + TailwindCSS）](/templates/next-tailwind)
- [Next.js（页面目录）](/templates/next-pages-dir)
- [Remix](/templates/remix)

<Step>2</Step> 在新项目的 <code>remotion/Root.tsx</code> 文件中，用现有项目的 <code>src/Root.tsx</code> 中的代码替换示例代码。递归包含所有其他组件，直到 <code>npx tsc -w</code> 不再报错。

<Step>3</Step> 将 <code>public</code> 文件夹的内容复制到新项目中。

<Step>4</Step> 找到 <code>{'<Player>'}</code> 渲染的位置：
<ul>
	<li>
		<strong>在 Next.js 页面目录模板中：</strong> 在 <code>pages/index.tsx</code> 下。
	</li>
	<li>
		<strong>在 Next.js 应用程序目录模板中：</strong> 在 <code>app/page.tsx</code> 下。
	</li>
	<li>
		<strong>在 Remix 模板中：</strong> 在 <code>app/routes/_index.tsx</code> 下。
	</li>
</ul>

选择您要在 Player 中呈现的主要 [组合](/docs/terminology/composition)，并在 <code>&lt;Player&gt;</code> 中包含其 <code>component</code> 和其他元数据。您可以从 <code>remotion</code> 文件夹导入组件，以便在 Studio 和应用程序中都导入。

如果要消除元数据的重复，您可以在单独的文件中定义常量，如 `DURATION_IN_FRAMES` 和 `FPS`，并在 Studio 和应用程序中导入这些常量。

如果您使用 [`calculateMetadata()`](/docs/dynamic-metadata#with-the-player)，请参阅[此处](/docs/dynamic-metadata#with-the-player)了解如何在 Player 中使用它。

<Step>5</Step> 如果您在 <code>remotion.config.ts</code> 中定义了任何 Webpack 覆盖，请查看如何将其应用到您的框架中。
