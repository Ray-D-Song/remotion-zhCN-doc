标准的 GCP Cloud Run 配额适用（[请参见此处](https://cloud.google.com/run/quotas)），其中最重要的是：

- **并发性**：默认情况下，Remotion Cloud Run 部署时并发性为 0。这意味着每个请求将获得自己的实例。但是，Remotion 团队尚未测试一个实例上的并发请求。
- **实例**：Cloud Run 服务实例的数量，可响应请求创建。可配置，最多限制为 100。可以通过 GCP 控制台请求更高的配额限制。
- **内存**：可配置，最多限制为 32GB
- **执行限制**：可配置，最多为 60 分钟

## 增加实例配额的步骤

导航至 [IAM 中的配额](https://console.cloud.google.com/iam-admin/quotas?service=run.googleapis.com&usage=ALL&project=_) 并选择您的 Remotion 项目。

您可以请求增加每个 GCP 区域的实例限制。选择所需的每个区域，确保“配额”列中的值为每个区域的实例限制。在右上角，选择“编辑配额”。

按照提示操作，使用以下要点作为指南。不能保证示例描述将适用于未来的请求。

- “每个区域的实例限制”配额是 Cloud Run Admin API 的一部分。
- 示例请求描述 - “希望确保我们的用户没有等待时间。我们正在运行的 Cloud Run 服务不能有任何并发性，因此我们依靠为多个请求启动额外的实例。”
- 示例预期用例 - “为用户渲染视频，使用 Remotion (https://remotion.dev)。对服务的每个请求都会渲染一个视频并将其存储在 Cloud Storage 中，然后完成。”
- 示例预期使用模式 - “我预计在澳大利亚的工作时间内会有突发情况。”
- 当被问及容器是否支持并发请求时，请输入 0。
- 我如何得出请求的实例数量的示例 - “希望将现有配额翻倍。”
