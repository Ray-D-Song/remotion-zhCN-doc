Remotion Cloud Run 支持**每个区域和项目一个存储桶**，您在 Remotion Cloud Run 中使用。不需要创建多个存储桶，因为：

- Cloud Storage 存储桶是一个有效的无限可扩展的存储解决方案。
- Remotion 将完全隔离每个渲染，以确保它们不会相互干扰。
- 您可以为站点提供一个唯一标识符，以区分生产环境和开发环境。
- Remotion Cloud Run 服务是一个二进制文件，不会随着您的代码库而更改。

您可能会直观地创建多个存储桶，因为您有多个环境，但通常不需要。

## 删除多余的存储桶

如果您收到以下错误消息

```
You have multiple buckets [remotioncloudrun-1a2b3c4d, remotioncloudrun-howDidThisHappen] in your Cloud Storage region [us-east1] starting with "remotioncloudrun-".
```

请在 GCP 控制台中删除多余的存储桶以修复错误。
