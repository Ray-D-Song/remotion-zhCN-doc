本文描述了 Remotion Cloud Run 所需的必要权限，并向感兴趣的人解释了为什么这些权限是必要的。

要了解如何设置权限的逐步指南，请[查看设置指南](/docs/cloudrun/setup)。

## 服务帐号权限

此策略应分配给 **Remotion 服务帐号**。在自动设置之后，通过创建一个名为 _Remotion API 服务帐号_ 的自定义角色，并将该角色分配给 Remotion 服务帐号来实现此目标。

要查看并手动编辑 GCP 项目中的角色，请转到 [IAM 管理员中的角色页面](https://console.cloud.google.com/iam-admin/roles)。

要查看并手动编辑分配给用户或服务帐号的权限/角色，请转到 [IAM 管理员中的 IAM 页面](https://console.cloud.google.com/iam-admin/iam)。

## 验证

有两种方法可以测试服务帐号的权限是否已正确设置。您可以执行以下命令之一：

```bash
npx remotion cloudrun permissions
```

或者，如果您想以编程方式验证它，请使用[`testPermissions()`](/docs/cloudrun/testpermissions)函数。
