## 作为一个包管理器

您可以使用 `bun i` 来初始化我们所有的 Remotion 模板。  
要使用 bun 创建一个新项目，请使用：

```
bun create video
```

此命令将所有脚本设置为使用 [`bunx remotionb`](/docs/cli/#using-bun)，这将使用 Bun 作为运行时。  
如果您想要使用 Node.js 作为运行时，请将 `remotionb` 更改为 `remotion`。

## 作为一个运行时

截至 Bun 1.0.24 和 Remotion 4.0.88，已知以下问题：

- ⚠️ `<Composition>` 和 `<Player>` 上的 `lazyComponent` 属性不起作用，此功能会自动禁用。
- ⚠️ 服务器端渲染脚本在运行完成后可能不会自动退出。

如果您发现更多问题，请随时向 Remotion 提交问题。

截至 Bun 1.0.24，此前列出的问题已解决。

## 对于贡献者

使用 `bun run start-bun` 启动示例测试平台。
