我们允许您在自 Remotion 2.6.5 起在 Chromium 和 Google Chrome 中设置以下标志：

## `--disable-web-security`

这将主要禁用 CORS 和其他安全功能。

:::note
Remotion 将自动附加 `--user-data-dir` 标志。
:::

### 通过 Node.JS API

在 [`getCompositions()`](/docs/renderer/get-compositions), [`renderStill()`](/docs/renderer/render-still#disablewebsecurity), [`renderMedia()`](/docs/renderer/render-media#disablewebsecurity), [`renderFrames()`](/docs/renderer/render-frames#disablewebsecurity), [`getCompositionsOnLambda()`](/docs/lambda/getcompositionsonlambda#disablewebsecurity), [`renderStillOnLambda()`](/docs/lambda/renderstillonlambda#disablewebsecurity) 和 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda#disablewebsecurity) 中，您可以传递 [`chromiumOptions.disableWebSecurity`](/docs/renderer/render-still#disablewebsecurity)。

### 通过 CLI 标志

在以下命令中之一中传递 [`--disable-web-security`](/docs/cli/render#--disable-web-security)：[`remotion render`](/docs/cli/render), [`remotion compositions`](/docs/cli/compositions), [`remotion still`](/docs/cli/still), [`remotion lambda render`](/docs/lambda/cli/render), [`remotion lambda still`](/docs/lambda/cli/still), [`remotion lambda compositions`](/docs/lambda/cli/compositions)。

### 通过配置文件

使用 [setChromiumDisableWebSecurity()](/docs/config#setchromiumdisablewebsecurity)。

```tsx twoslash
import {Config} from '@remotion/cli/config';

// ---cut---

Config.setChromiumDisableWebSecurity(true);
```

:::note
在 `v3.3.39` 之前，该选项称为 `Config.Puppeteer.setChromiumDisableWebSecurity()`。
:::

## `--ignore-certificate-errors`

导致无效的 SSL 证书（如自签名证书）被忽略。

### 通过 Node.JS API

在 [`getCompositions()`](/docs/renderer/get-compositions), [`renderStill()`](/docs/renderer/render-still#ignorecertificateerrors), [`renderMedia()`](/docs/renderer/render-media#ignorecertificateerrors), [`renderFrames()`](/docs/renderer/render-frames#ignorecertificateerrors), [`getCompositionsOnLambda()`](/docs/lambda/getcompositionsonlambda#disablewebsecurity), [`renderStillOnLambda()`](/docs/lambda/renderstillonlambda#ignorecertificateerrors) 和 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda#ignorecertificateerrors) 中，您可以传递 [`chromiumOptions.ignoreCertificateErrors`](/docs/renderer/render-still#ignorecertificateerrors)。

### 通过 CLI 标志

在以下命令中之一中传递 [`--ignore-certificate-errors`](/docs/cli/render#--ignore-certificate-errors)：[`remotion render`](/docs/cli/render), [`remotion compositions`](/docs/cli/compositions), [`remotion still`](/docs/cli/still), [`remotion lambda render`](/docs/lambda/cli/render), [`remotion lambda still`](/docs/lambda/cli/still), [`remotion lambda compositions`](/docs/lambda/cli/compositions).

### 通过配置文件

使用 [setChromiumIgnoreCertificateErrors()](/docs/config#setchromiumignorecertificateerrors).

```tsx twoslash
import {Config} from '@remotion/cli/config';

// ---cut---

Config.setChromiumIgnoreCertificateErrors(true);
```

:::note
在 `v3.3.39` 之前，该选项称为 `Config.Puppeteer.setChromiumIgnoreCertificateErrors()`.
:::

## `--disable-headless`

<Options id="disable-headless" />

### 通过 Node.JS APIs

在 [`getCompositions()`](/docs/renderer/get-compositions), [`renderStill()`](/docs/renderer/render-still#headless), [`renderMedia()`](/docs/renderer/render-media#headless) 和 [`renderFrames()`](/docs/renderer/render-frames#headless) 中，您可以传递 [`chromiumOptions.headless`](/docs/renderer/render-still#headless). 无法在 Lambda 中设置此选项.

### 通过 CLI 标志

在以下命令中之一中传递 [`--disable-headless`](/docs/cli/render#--disable-headless)：[`remotion compositions`](/docs/cli/compositions), [`remotion render`](/docs/cli/render), [`remotion still`](/docs/cli/still).

### 通过配置文件

使用 [setChromiumHeadlessMode()](/docs/config#setchromiumheadlessmode).

```tsx twoslash
import {Config} from '@remotion/cli/config';

// ---cut---

Config.setChromiumHeadlessMode(false);
```

:::note
在 `v3.3.39` 之前，该选项称为 `Config.Puppeteer.setChromiumHeadlessMode()`.
:::

## `--gl`

<Options id="gl" />

### 通过 Node.JS APIs

在 [`getCompositions()`](/docs/renderer/get-compositions#chromiumoptions), [`renderStill()`](/docs/renderer/render-still#gl), [`renderMedia()`](/docs/renderer/render-media#gl), [`renderFrames()`](/docs/renderer/render-frames#gl), [`getCompositionsOnLambda()`](/docs/lambda/getcompositionsonlambda#gl), [`renderStillOnLambda()`](/docs/lambda/renderstillonlambda#gl) 和 [`renderMediaOnLambda()`](/docs/lambda/rendermediaonlambda#gl), 您可以传递 [`chromiumOptions.gl`](/docs/renderer/render-still#gl).

### 通过 CLI 标志

在以下命令中之一中传递 [`--gl=swiftshader`](/docs/cli)：[`remotion render`](/docs/cli/render), [`remotion compositions`](/docs/cli/compositions), [`remotion still`](/docs/cli/still), [`remotion lambda render`](/docs/lambda/cli/render), [`remotion lambda still`](/docs/lambda/cli/still), [`remotion lambda compositions`](/docs/lambda/cli/compositions).

### 通过配置文件

```tsx twoslash
import {Config} from '@remotion/cli/config';

// ---cut---

Config.setChromiumOpenGlRenderer('swiftshader');
```

:::note
在 `v3.3.39` 之前，该选项称为 `Config.Puppeteer.setChromiumOpenGlRenderer()`。
:::
