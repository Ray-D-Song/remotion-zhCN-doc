并发性在不同的上下文中可能有不同的含义。

**在本地渲染中，** 并发性指的是在渲染过程中同时打开的浏览器选项卡数量。  
每个 Chrome 选项卡都会渲染网页内容，然后对其进行截图。

**对于在 Lambda 上进行渲染，** 并发性指的是为渲染生成的 Lambda 函数数量。您可以使用 [`framesPerLambda`](/docs/lambda/rendermediaonlambda#framesperlambda) 选项来控制 Lambda 的并发性。

通常每个 Lambda 函数只会打开一个选项卡，除非您使用 [`concurrencyPerLambda`](/docs/lambda/rendermediaonlambda#concurrencyperlambda) 选项来增加每个 Lambda 函数的选项卡数量。

更高的并发性可以加快渲染时间，但是过高的并发性会导致收益递减，并且会导致机器过载，可能会导致渲染崩溃。
