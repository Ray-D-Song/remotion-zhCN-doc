## 渲染过程中

Remotion 在渲染视频时执行算法来确定传递给组件的 props。有三个因素起作用：

<Step>1</Step> <strong>默认 props</strong> 是在渲染时传递给组件的数据。您可以使用您的 <a href="/docs/composition#defaultprops"><code>defaultProps</code></a> 属性指定它们。<a href="/docs/composition#defaultprops"><code>&lt;Composition /&gt;</code></a>。<br/>

<Step>2</Step> <strong>输入 props</strong> 是在调用渲染时传递的数据，可以通过 <a href="/docs/renderer/render-media#inputprops"><code>inputProps</code></a> 选项、<a href="/docs/cli/render#--props"><code>--props</code> flag</a> 或使用 Remotion Studio 中的渲染对话框传递。<br/>

<Step>3</Step> 可以使用 <a href="/docs/composition#calculatemetadata"><code>calculateMetadata()</code></a> 动态转换 props，以及组合的元数据。

## 在 Remotion Studio 中

在 Remotion Studio 中，props 的解析方式类似，但有一些<strong>不同之处</strong>：

<Step>1</Step> 默认 props 可以在右侧边栏中进行<strong>编辑</strong>。无效的修改将被标记为红色轮廓，不会应用。<br/>
<Step>2</Step> 如果您使用渲染按钮渲染视频，则<strong>输入 props 表单会预先填充</strong>默认 props，包括右侧边栏中的修改。<br/><br/>

以下规则<strong>保持不变</strong>，您应该注意：

<Step>1</Step> 如果您使用 <a href="/docs/cli/studio#--props"><code>--props</code></a> flag 启动 Studio，则此数据将优先于默认 props，包括右侧边栏中的修改。不建议将输入 props 传递给 Studio。<br/>

<Step>2</Step> 传入的输入属性可能会被 <a href="/docs/composition#calculatemetadata"><code>calculateMetadata()</code></a> 转换。 <br/>
