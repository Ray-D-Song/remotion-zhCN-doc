用于计算播放器大小的算法如下：

<Step>1</Step> 默认情况下，播放器的大小与由<code>compositionHeight</code>和<code>compositionWidth</code>属性定义的合成高度一样大。<br/>
<Step>2</Step> 如果使用<code>style</code>属性定义了<code>height</code>和<code>width</code>，播放器将采用您传递的尺寸。<br/>
<Step>3</Step> 如果使用<code>style</code>属性传递了<code>height</code>，播放器将采用该高度，并根据视频的宽高比计算宽度。<br/>
<Step>4</Step> 如果使用<code>style</code>属性传递了<code>width</code>，播放器将采用该宽度，并根据视频的宽高比计算高度。<br/><br/>

:::note
在v3.3.43之前，如果发生<InlineStep>3</InlineStep>或<InlineStep>4</InlineStep>的情况，由于元素被测量，挂载过程中会发生布局移位。使用更新版本的Remotion将解决此问题，因为它使用`aspect-ratio` CSS属性。
:::

## 全宽度

通过设置以下样式：

```tsx
style={{ width: "100%" }}
```

视频将按照父容器的全宽度进行缩放，而高度将根据视频的宽高比进行计算。

## 适应容器

您可以使用

```tsx
style={{ width: "100%", height: "100%" }}
```

使`<Player />`适应整个容器。请注意，如果播放器大小与视频宽高比不匹配，视频将被缩小并居中显示在播放器中。
