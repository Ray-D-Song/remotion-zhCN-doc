## 语言指南

- **保持简洁**：开发人员不喜欢阅读，因此添加过多的文字会导致信息丢失。
- **链接术语**：在使用 Remotion 特定术语时，请链接到[术语表](/docs/terminology)页面进行解释。
- **避免情感**和填充语言（"太好了！让我们继续下一步"）。通常可以删除而不会丢失任何信息。
- **如果某个部分变得太长，请分成多个段落**。
- **将读者称为"您"**，而不是"我们"。
- **不要责怪用户**：而不是说"您提供了错误的输入"，请说"输入无效"或"提供了错误的输入"。
- **不要假设很容易**：避免使用"只需"和"仅仅"这样的词语，因为初学者可能不会觉得这么简单。

## 添加代码片段

您可以通过在段落开头使用三个反引号<code>```</code>来添加代码片段。代码将根据您在反引号后指定的语言进行高亮显示。
