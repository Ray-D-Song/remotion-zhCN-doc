一些问题带有现金悬赏，您可以通过解决它们来赚取奖金。

<Bounties />

## 查找悬赏

所有悬赏都可以在 Remotion 的 GitHub Issues 中跟踪。带有悬赏的问题上有一个标签 `💎 Bounty`。 <br/>

- [查看开放的悬赏](https://github.com/remotion-dev/remotion/issues?q=is%3Aopen+label%3A%22%F0%9F%92%8E+Bounty%22+sort%3Aupdated-desc) <br/>
- [查看已完成的悬赏](https://github.com/remotion-dev/remotion/issues?q=label%3A%22%F0%9F%92%8E+Bounty%22+sort%3Aupdated-desc+is%3Aclosed) <br/>

悬赏也会在 <a href="https://remotion.dev/discord">Remotion Discord 服务器</a> 的 `#bounties` 频道中宣布。

## 接受悬赏

与普通问题一样，您可以通过评论来表示您愿意解决问题。对于带有悬赏的问题，Remotion 团队会快速筛选以查看经验是否与问题难度匹配。

分配是按照先到先得的原则进行的。

同一贡献者一次只能分配一个悬赏。

## 开始解决问题之前

悬赏是通过第三方平台（即 Algora）支付的。

在开始解决问题之前，建议您在 <a href="https://algora.io/">Algora</a> 上创建一个帐户，在那里您还需要设置并连接您的 Stripe 帐户。一旦您被分配了问题并在 Algora 上创建了帐户，您就可以开始了。

## 开始攻击

假定您在分配问题后的合理时间内开始解决问题。卡住了不是问题，但您应该进行沟通。通过讲述和描述挑战，您可能会得到社区的一些意见，这有助于您更快地解决问题。

如果您一周内没有更新，并且看起来问题被放弃了，我们保留将问题重新分配给其他人的权利。

## 获取帮助



在[Hacktoberfest](https://github.com/remotion-dev/remotion/issues?q=label%3A%22%F0%9F%A5%A8++hacktoberfest%22+sort%3Aupdated-desc+is%3Aopen)期间（即十月），Remotion团队提供更多指导，因为这是帮助首次贡献者的活动。

在一年的其余时间里，处理悬赏问题时提供的指导较少。您应该对如何解决问题有基本的了解。当然，总会有问题出现，但与Hacktoberfest期间相比，提供的帮助较少。

## 验收标准

每个悬赏问题都有一个描述预期交付内容的描述。每次，还期望以下内容：

- 新选项或功能的文档
- 测试和格式的绿色勾选
- 当您将PR标记为准备就绪时，不应保留`console.log`和注释掉的代码
