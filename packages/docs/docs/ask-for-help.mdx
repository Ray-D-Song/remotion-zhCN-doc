要获得帮助，请执行以下操作：

<Step>1</Step> 在 <a href="https://remotion.dev/issue">remotion.dev/issue</a> 下提交问题报告 <br/>
<Step>2</Step> 在 <a href="https://remotion.dev/discord">remotion.dev/discord</a> 的 <code>#help</code> 频道或 <code>#help-forum</code> 中写入

## 提供信息

提供与问题相关的信息。您提供的信息越多，我们就能提供的帮助就越多！

**当您遇到错误时：**

<Step>1</Step> 粘贴完整的错误消息，包括堆栈跟踪。 <br/>
<Step>2</Step> 提供导致此错误的代码或命令。 <br/>

<br/>

**当您没有收到错误时：**

<Step>1</Step> 提供您执行的代码或命令。 <br/>
<Step>2</Step> 描述您的预期结果。 <br/><br/>

始终提及 Remotion 版本：

```
npx remotion versions
```

## 自行调查

以下信息对我们也可能很有用：

<Step>1</Step> 您已经尝试了什么？<br/>
<Step>2</Step> 您是否怀疑可能导致问题的原因？<br/>
<Step>3</Step> 如果删除某些代码，问题是否会消失？<br/>
<Step>4</Step> 您的堆栈或操作系统中是否有任何异常部分？

## 做

<Step>1</Step> 使用搜索引擎或 ChatGPT 解决特定于 React 的问题。 <br/>
<Step>2</Step> 搜索超过 1100+ <a href="https://github.com/remotion-dev/remotion/issues">GitHub 问题</a> 和 400+ <a href="/search">文档页面</a>。 <br/>
<Step>3</Step> 通过缩小代码范围尝试找出问题的根本原因。<br/>
<Step>4</Step> 提供相关的代码片段或重现。

## 不要

<Step error>1</Step> 使用电子邮件或直接消息寻求支持。 <br/>
<Step error>2</Step> 在未先 <a href="/search">搜索</a> 文档的情况下提问。 <br/>

## 付费支持

我们的 [Remotion 专家](/experts) 可提供技术建议和指导。凭借他们在创建 Remotion 项目方面的丰富经验，他们一定会提供极大的帮助。

您还可以在此处向 Remotion 的创始人请求咨询会话 [here](https://cal.com/remotion/consulting)。  
这是唯一通过视频通话从 Remotion 团队获得支持的方式。
