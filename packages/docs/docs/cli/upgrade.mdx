升级所有与 Remotion 相关的包。

```
npx remotion upgrade
```

## 标志

### `--package-manager`<AvailableFrom v="3.2.33"/>

_可选_

强制使用特定的包管理器。如果您在 monorepo 中使用 Remotion 并希望一次升级所有包，则这很有用。默认情况下，Remotion 将自动检测包管理器。

可接受的值为 `npm`、`yarn` 和 `pnpm`

### `--version`<AvailableFrom v="4.0.15"/>

安装特定版本。还可以降级到旧版本。

## 包管理器支持

`npm`、`yarn` 和 `pnpm` 都受支持。

## 与 `npm update`、`yarn upgrade`、`pnpm up` 的区别

这些命令在没有参数的情况下执行时将升级项目中的所有依赖项。我们建议不要这样做，因为您可能会无意中破坏项目的其他部分，而您只想升级 Remotion。
