根据入口点、合成 ID 渲染视频或音频，并将其保存到输出位置。

```bash
npx remotion render <entry-point|serve-url>? <composition-id> <output-location>
```

您可以将 [Serve URL](/docs/terminology/serve-url) 或 [入口点](/docs/terminology/entry-point) 作为第一个参数传递，否则入口点将会被 [确定](/docs/terminology/entry-point#which-entry-point-is-being-used)。

如果未传递 `composition-id`，Remotion 将要求您选择一个合成。  
如果未传递 `output-location`，媒体将被渲染到 `out` 文件夹中。

## 标志

除了使用命令行参数选择视频和输出位置外，还支持以下标志：

### `--props`

[传递给视频所选合成的输入属性。](/docs/passing-props#passing-input-props-in-the-cli)  
必须是一个序列化的 JSON 字符串 (`--props='{"hello": "world"}'`) 或一个指向 JSON 文件的路径 (`./path/to/props.json`)。  
从根组件中，可以使用 [`getInputProps()`](/docs/get-input-props) 读取属性。  
您可以使用 [`calculateMetadata()`](/docs/calculate-metadata) 转换输入属性。

:::note
在 Windows shell 上不支持内联 JSON 字符串，因为它会移除 `"` 字符，请改用文件名。
:::

### `--height`<AvailableFrom v="3.2.40" />

[覆盖合成高度。](/docs/config#overrideheight)

### `--width`<AvailableFrom v="3.2.40" />

[覆盖合成宽度。](/docs/config#overridewidth)

### `--concurrency`

[使用多少 CPU 线程。](/docs/config#setconcurrency) 最小为 1。最大值为您拥有的线程数量（在 Node.JS 中为 `os.cpus().length`）。您还可以提供一个百分比值（例如 50%）。

### `--pixel-format`

[设置自定义像素格式。请查看此处可用的值。](/docs/config#setpixelformat)

### `--image-format`<AvailableFrom v="1.4.0" />

[`jpeg` 或 `png` - JPEG 更快，但不支持透明度。](/docs/config#setvideoimageformat) 默认图像格式自 v1.1 起为 `jpeg`。

### `--config`<AvailableFrom v="1.2.0" />

指定 Remotion 配置文件的位置。

### `--env-file`<AvailableFrom v="2.2.0" />

指定 dotenv 文件的位置。默认为 `.env`。

### `--jpeg-quality`<AvailableFrom v="4.0.0" />

[JPEG 渲染质量介于 0 和 100 之间](/docs/config#setjpegquality)。在渲染 PNG 帧时不起作用。

### ~~`--quality`~~<AvailableFrom v="1.4.0" />

在 v4.0.0 中更名为 `--jpeg-quality`。

### `--output` <AvailableFrom v="4.0.0" />

设置输出文件路径，作为 `output-location` 位置参数的替代。

### `--overwrite`

[即使文件已存在也写入输出。](/docs/config#setoverwriteoutput) 默认情况下启用此标志，使用 `--overwrite=false` 来禁用它。

### `--sequence`<AvailableFrom v="1.4.0" />

[如果要将图像序列作为输出而不是视频，请传递此标志。](/docs/config#setimagesequence)

### `--codec`<AvailableFrom v="1.4.0" />

[`h264` 或 `h265` 或 `png` 或 `vp8` 或 `vp9` 或 `mp3` 或 `aac` 或 `wav` 或 `prores` 或 `h264-mkv`](/docs/config#setcodec)。如果不提供 `--codec`，将使用 H.264 编码器。

### `--crf`<AvailableFrom v="1.4.0" />

[设置输出的 Constant Rate Factor (CRF)](/docs/config#setcrf)。最小为 0。如果希望保持最佳质量并较少关心文件大小，请使用此速率控制模式。如果设置了 `--video-bitrate`，则无法设置此选项。

### `--browser-executable`<AvailableFrom v="1.5.0" />

[Chrome可执行文件的路径](/docs/config#setbrowserexecutable)。如果未指定并且Remotion找不到可执行文件，则在渲染过程中将下载一个。

### `--scale`

[按照传入的因子缩放输出帧。](/docs/scaling) 例如，一个 1280x720px 的帧将以 `1.5` 的缩放因子变为 1920x1080px 的帧。像字体和 HTML 标记这样的矢量元素将以更多细节呈现。`scale` 必须大于 0 且小于等于 16。默认值：`1`。

### `--frames`<AvailableFrom v="2.0.0" />

[渲染视频的子集](/docs/config#setframerange)。例如：`--frames=0-9` 选择前 10 帧。要渲染静帧，请使用 `still` 命令。

### `--every-nth-frame`<AvailableFrom v="3.1.0" />

[仅渲染每第 n 帧。](/docs/config#seteverynthframe) 此选项仅在渲染 GIF 时可设置。这允许您降低 GIF 的帧速率。

例如，仅每隔一帧、每隔三帧等。仅适用于渲染 GIF。[更多详情请参见此处。](/docs/render-as-gif#reducing-frame-rate)

### `--muted`<AvailableFrom v="3.2.1" />

[禁用音频输出。](/docs/cli/render#--muted) 此选项仅在渲染视频时可用。

### `--enforce-audio-track`<AvailableFrom v="3.2.1" />

[如果没有音频轨道，则渲染一个静音音轨。](/docs/cli/render#--enforce-audio-track)。

定义单个帧在解析所有 [`delayRender()`](/docs/delay-render) 调用之前的超时时间（以毫秒为单位）。默认值：`30000`。

:::info

:::
