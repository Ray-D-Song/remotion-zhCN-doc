欢迎阅读有关在云上使用 Remotion 渲染的发布说明！通过 Remotion 云运行，您现在可以轻松在 Google Cloud Platform 中进行渲染。

## 简化的项目设置

在设置 Remotion Lambda 后，经常会看到开发人员在几次误点击后需要在 [Discord](https://remotion.dev/discord) 上寻求帮助。对于 Cloud Run，在控制台中只需点击几下，然后 Terraform 接管自动化设置过程 - 启用 API、创建具有所需权限的角色，并创建一个服务帐号以安全地执行渲染。它甚至会创建您的 .env 文件，可直接插入到您的 remotion 项目中！

## 强大的渲染

目前，Remotion 云运行仅支持单线程渲染过程。Remotion Lambda 可以通过启动多个函数并并行渲染块，然后将它们拼接在一起来提供更快的速度。不过，这种功能通常会牺牲可靠性，根据您正在构建的产品，可靠性可能比速度更重要。

## 更容易理解的并发性 + 实例限制

Lambda 是否具有零并发性？

在使用 AWS Lambda 时，您可能会遇到允许多个用户同时渲染视频的情况，并需要计算您的帐户上是否还有并发性以允许另一个渲染。Remotion Lambda 会为并行渲染块启动多个 Lambda，这需要考虑到同时渲染请求。在 Remotion 云运行中，您可以轻松将实例限制设置为 100（默认值），并且知道如果有 99 个用户同时请求渲染，您仍然有一个服务实例可用。

## 更高的配额限制



在撰写本文时，以下是GCP和AWS的配额差异，以及这对您的产品可能意味着什么：

| 约束             | AWS 限制   | GCP 限制   | 结果                                                                                                                                                                                                                                                                                                                                                                                    |
| ----------------- | ---------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 磁盘大小         | 10 GB      | 32 GB      | 在GCP中，您可以使用更大的文件大小渲染视频。文件大小随着质量和视频长度的组合而增加。由于分布式渲染需要（一般而言）在将所有块拼接在一起之前复制所有块，Lambda在这里受到进一步限制。[这将使Lambda的输出降至5GB](/docs/lambda#limitations)。                                        |
| 最小实例数       |            |            |                                                                                                                                                                                                                                                                                                                                                                                            |
| 最大超时时间     | 15 分钟    | 60 分钟    | 由于Lambda与Cloud Run上的分布式渲染的性质不同，这很难进行比较。由于Cloud Run在一个实例中执行所有渲染，这允许整个渲染过程最多持续60分钟。对于Lambda，每个块的最大超时时间为15分钟，但执行连接操作的函数也需要在15分钟内完成。 |
