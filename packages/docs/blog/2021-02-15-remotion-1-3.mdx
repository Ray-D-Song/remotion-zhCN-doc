距离初始发布仅1周，Remotion 1.3来了！仅仅7天，我们合并了超过40个拉取请求 - 简直太神奇了！让我们看看这个版本的变化。

## 速度提升2倍 - 再次！

2天前，渲染时间减半，而在这个版本中，我们又将其减半了！查看这个[Spotify Wrapped](https://github.com/JonnyBurger/remotion-wrapped)示例的基准测试：

Remotion 1.1

```
hyperfine --min-runs 5  'npm run build -- --overwrite --concurrency=16'
Benchmark #1: npm run build -- --overwrite --concurrency=16
  Time (mean ± σ):     98.972 s ±  0.650 s    [User: 123.329 s, System: 10.103 s]
  Range (min … max):   97.951 s … 99.540 s    5 runs
```

Remotion 1.3

```
hyperfine --min-runs 5  'npm run build -- --overwrite --concurrency=16'
Benchmark #1: npm run build -- --overwrite --concurrency=16
  Time (mean ± σ):     17.921 s ±  0.224 s    [User: 36.492 s, System: 3.482 s]
  Range (min … max):   17.650 s … 18.264 s    5 runs
```

从98秒到18秒 - 快了5.5倍！同时，我们达到了一个重要的里程碑：这个19秒长的720p视频比实时渲染速度更快。尽管我的电脑比大多数电脑更快（8核Intel i9-9900K芯片），但仍然非常令人印象深刻！

我们通过各种Puppeteer渲染管道优化实现了这一性能提升。特别感谢[jeetiss](https://github.com/jeetiss)实现了一项复杂的性能优化，不再需要为每一帧重新加载页面。

## 支持纯JavaScript

我的目标是强制每个人都使用Typescript - 但我失败了。现在已经添加了对纯JavaScript的支持！查看[这里](/docs/javascript)如何启用它。请小心操作 🙈

## 类型安全的配置文件

许多您可以通过CLI标志传递的选项，现在也可以通过在存储库中添加`remotion.config.ts`文件来添加。例如，如果您想将并发性增加到您拥有的线程数量，并且永远不想写`--overwrite`，您可以将以下内容添加到配置文件中：

```tsx
import os from "os";
import { Config } from "remotion";

Config.Rendering.setConcurrency(os.cpus().length);
Config.Output.setOverwriteOutput(true);
```

您可以在[此页面上](https://www.remotion.dev/docs/config)查看所有选项。使用Typescript创建配置文件的目标是提供自动完成功能，轻松突出显示已弃用的选项，并使未来更改选项时更容易展示如何迁移。

## 缓动 API

尽管存在缓动 API，但它是未记录的。了解可用的[Easing 方法](/docs/easing)以及如何与[`interpolate()`](/docs/interpolate)一起使用！

```tsx
import { Easing, interpolate } from "remotion";

interpolate(frame, [0, 100], {
  easing: Easing.bezier(0.8, 0.22, 0.96, 0.65),
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp",
});
```

## 一些小事

- Remotion 的导出已经整理过，并且内部内容已经移动到 `Internals` 中。您不应该依赖这些 API，因为它们可能随时间变化而发生变化。

- [现在捆绑步骤有了进度指示器。](https://github.com/remotion-dev/remotion/pull/83)

- [您现在可以配置一个不同的像素格式，该格式将传递给 FFMPEG](/docs/config#setpixelformat)。

- [现在支持类属性。](https://github.com/remotion-dev/remotion/pull/95)

- [PNG 序列现在具有填充的文件名（`element-0000.png` 而不是 `element-0.png`），以便更好地与 ImageMagick 进行互操作，并确保文件资源管理器中的正确顺序](https://github.com/remotion-dev/remotion/pull/91)

- [当 FFMPEG 在 Windows 上没有正确链接时，会提供更好的错误消息](https://github.com/remotion-dev/remotion/pull/98)

- [修复了很多](https://github.com/remotion-dev/remotion/pull/92) [拼写](https://github.com/remotion-dev/remotion/pull/93) [错误](https://github.com/remotion-dev/remotion/pull/85)！

- [为文档添加了分析](https://github.com/remotion-dev/remotion/pull/78) - 您可以在[这里](https://app.splitbee.io/public/remotion.dev)查看 Remotion 文档的页面浏览量。

- [修复了 PowerShell 的问题](https://github.com/remotion-dev/remotion/pull/97)

- [修复了一个 puppeteer 问题 "Session closed. Most likely the page has been closed"](https://github.com/remotion-dev/remotion/issues/69)。

- 从存储库中删除了大文件，因此克隆速度更快。
